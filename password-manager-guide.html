Вопрос-ответ

Частые вопросы

Vaultwarden и Bitwarden — это одно и то же?
Клиентская часть — да. Vaultwarden реализует тот же API что и официальный Bitwarden сервер, поэтому все официальные приложения Bitwarden (iOS, Android, браузерные расширения, десктоп) работают с Vaultwarden без изменений. Разница в серверной части: Vaultwarden написан на Rust сообществом, потребляет в 40 раз меньше RAM, но не имеет официальных security аудитов и SOC 2 сертификации.
Какое минимальное железо нужно?
Vaultwarden: любой VPS с 512MB RAM, даже Raspberry Pi. Passbolt: минимум 2GB RAM, рекомендуется 4GB. Psono: от 512MB. Bitwarden официальный: минимум 2GB, рекомендуется 4GB+. Для команды до 50 человек достаточно VPS с 2 ядрами CPU и 2GB RAM для любого из этих инструментов.
Как интегрировать с Active Directory / LDAP?
Psono — LDAP доступен в Community Edition бесплатно. Passbolt — LDAP только в Pro версии ($49+/мес). Bitwarden официальный — LDAP в Enterprise плане. Vaultwarden — нет нативного LDAP, но можно настроить через обратный прокси с LDAP аутентификацией.
Насколько сложен деплой?
Vaultwarden: один Docker контейнер, 5 минут. Psono: Docker Compose, 15–30 минут. Passbolt: Docker или пакетная установка, 30–60 минут (требует настройки GPG ключей). Bitwarden официальный: Docker Compose с несколькими сервисами, 1–2 часа. Всё документировано, есть официальные гайды.
Что делать с бэкапами?
Критически важно настроить автоматические бэкапы до начала использования. Vaultwarden: бэкап всей директории данных + SQLite файл, ежедневно. Passbolt: дамп MySQL + GPG ключи сервера. Psono: дамп PostgreSQL/MySQL. Бэкапы должны быть зашифрованы и храниться off-site. 30% команд бросают self-hosted решения именно из-за потери данных без бэкапов.
Можно ли мигрировать с одного решения на другое?
Да. Все перечисленные инструменты поддерживают импорт/экспорт через CSV и JSON. Bitwarden/Vaultwarden экспортируют в формат который понимает Passbolt и Psono. Passwork поддерживает bulk-импорт с сохранением иерархии папок. Рекомендуется тестовая миграция на staging до переноса продакшен данных.

Безопасность, сравнение шифрования, советы по настройке.

Как устроено шифрование в каждом решении
Vaultwarden / Bitwarden

AES-256-CBC + PBKDF2 (или Argon2). Мастер-пароль никогда не покидает устройство. На сервере хранятся только зашифрованные данные.

Passbolt

OpenPGP end-to-end шифрование на каждый пароль отдельно. Сервер физически не может расшифровать данные — только клиент с приватным ключом.

Psono

libsodium (NaCl) — современная криптография. Клиентское шифрование перед отправкой на сервер.

Советы по безопасной настройке self-hosted
1. VPN или закрытая сеть

Не выставляйте менеджер паролей в открытый интернет без крайней необходимости. Доступ через VPN (WireGuard/Tailscale) значительно снижает поверхность атаки.

2. HTTPS обязательно

Let's Encrypt через nginx/Caddy. Без HTTPS браузерные расширения откажутся подключаться.

3. 2FA для всех пользователей

TOTP обязателен. Passbolt и Psono позволяют принудительно включить 2FA для всех пользователей через admin политики.

4. Автоматические бэкапы с первого дня

Ежедневный зашифрованный бэкап базы данных и конфигурации. Хранить минимум в двух местах. Тестировать восстановление раз в месяц.
